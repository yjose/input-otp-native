`focusSlot(index)` is an imperative method on `OTPInputRef` that truncates the
value to `index` characters and focuses the input — making slot `index` the new
active slot. This lets users tap any slot directly to resume typing from there.

## How it works

Wrap every slot in a `Pressable` and call `ref.current?.focusSlot(index)` in
its `onPress`. The library handles the rest:

- Truncates the current value to `index` characters
- Focuses the underlying `TextInput`
- The standard sequential-input logic takes over from that slot

## Edge cases

| Call                               | Result                                        |
| ---------------------------------- | --------------------------------------------- |
| `focusSlot(0)`                     | Clears all digits (equivalent to `clear()`)   |
| `focusSlot(maxLength)`             | Focuses the end without clearing any digits   |
| `focusSlot(-1)` or `focusSlot(99)` | Clamped to `[0, maxLength]` automatically     |
